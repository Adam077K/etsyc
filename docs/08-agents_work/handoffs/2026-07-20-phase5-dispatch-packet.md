## A2 ¬∑ Cross-cutting conventions (ALL 32 files)

1. **Template is law.** Fill `_TEMPLATE.md` top-to-bottom: Metadata ‚Üí Prioritization (RICE + MoSCoW) ‚Üí Overview ‚Üí Problem ‚Üí Proposed Solution (numbered UX flow) ‚Üí User Stories ‚Üí Acceptance Criteria (Given/When/Then) ‚Üí UX/UI Notes ‚Üí Technical Requirements ‚Üí NFRs ‚Üí Dependencies ‚Üí Out of Scope ‚Üí Risk Assessment ‚Üí Success Metrics ‚Üí Rollout Plan ‚Üí Open Questions ‚Üí Changelog. Delete a genuinely-N/A section only with a one-line reason; never leave placeholder text. Status=`Draft`; Author=`CPO (Phase-5 spec worker)`; Reviewers=`CPO + CTO`.
2. **D#-traceability (hard gate).** Overview cites the exact D#. ACs trace to a locked decision or named engine/schema contract. Behavior with no backing D#/contract ‚Üí Open Questions + escalate; do NOT invent scope.
3. **All-4-states mandatory.** Any UI spec defines **empty ¬∑ loading ¬∑ error ¬∑ success** in UX/UI Notes AND covers them in ACs. *empty ‚âÝ blank* (live worlds omit truly-empty optional blocks; seller preview shows guiding prompts tied to the interview beat); *loading = skeleton matched to real layout*, never a centered spinner; *error = quiet + inline + recoverable*, never blocking the rest of the world; *success* = the designed happy state. `success`-only ACs are not shippable.
4. **Brand / anti-slop rails (D9 + D15) ‚Äî state which surface you touch:**
   - **KOL's own product UI** (feed, nav, chrome, checkout, marketing, dashboard) ‚Üí FIXED curated system: `theme.kind:"curated"` enums only.
   - **Seller shops** ‚Üí FULL brand freedom (`theme.kind:"custom"`, D15). Anti-slop for shops is NOT palette-capping; it is the deterministic WCAG-AA gate + auto-critic (P9) + maker approval (P10). Publish blocked until AA-PASS ‚àß all rendered blocks approved.
